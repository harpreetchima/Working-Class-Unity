# HeroUI  Preline UI

Looking for a [HeroUI alternative](https://daisyui.com/alternative/heroui/)? This page compares HeroUI
and Preline UI, two popular UI component libraries. We are comparing features, size,
efficiency and usage data to help you choose which component library is better for your next
project.

| HeroUI | Preline UI |
| --- | --- |
| MIT<br>License<br>Open source MIT License | MIT<br>License<br>Open source MIT License |
| Only React<br>Frameworks<br>HeroUI works only with React | ALL<br>Frameworks<br>Preline is framework agnostic and works everywhere |
| 43<br>Unique components<br>HeroUI has 50 Components, 43 of them are unique – for example pagination and button group are considered as same. | 66<br>Unique components<br>Preline has 77 Components, 66 of them are unique – for example pagination and button group are considered as same. |
| 2<br>Built-in Themes<br>HeroUI has light and dark themes | 2<br>Built-in Themes<br>Preline has light and dark themes |
| No<br>Supports more than two themes<br>Does not support more than 2 themes at the same time | No<br>Supports more than two themes<br>Does not support more than 2 themes at the same time |
| 1.2MB<br>JavaScript size<br>Imported JavaScript bundle size (minified) | 84.6kB<br>JavaScript size<br>Imported JavaScript bundle size (minified) |
| 208<br>Dependencies<br>HeroUI has 208 third-party dependencies. Using more third-party dependencies increases the risk of security vulnerabilities, compatibility problems, and long-term maintenance headaches from deprecated packages. | 3<br>Dependencies<br>Preline has 3 third-party dependencies. |
| 37MB<br>Dependency size | 400kB<br>Dependency size |
| 23800<br>GitHub stars | 5400<br>GitHub stars |
| 71000<br>Used by open source projects<br>Based on GitHub's public repositories | 12700<br>Used by open source projects<br>Based on GitHub's public repositories |

| No<br>CDN<br>HeroUI CSS file is not available on CDN | Yes<br>CDN<br>Preline CSS file is available on CDN |
| No<br>Semantic class names<br>HeroUI does not use semantic CSS class names. Using semantic and descriptive CSS class names improves readability and maintainability of code independent of the current framework. | No<br>Semantic class names<br>Preline does not use semantic CSS class names. Using semantic and descriptive CSS class names improves readability and maintainability of code independent of the current framework. |
| No<br>Global customizations<br>HeroUI requires changing JavaScript props one by one at build time to do design token customization | No<br>Global customizations<br>Preline requires changing class names one by one at build time to do customization |

## Install daisyUI

1\. Install daisyUI as a Node package:

NPM

```
npm i -D daisyui@latest
```

PNPM

```
pnpm add -D daisyui@latest
```

Yarn

```
yarn add -D daisyui@latest
```

Bun

```
bun add -D daisyui@latest
```

Deno

```
deno i -D npm:daisyui@latest
```

2\. Add daisyUI to app.css:

```
@import "tailwindcss";
@plugin "daisyui";
```

[Install guide](https://daisyui.com/docs/install/)
